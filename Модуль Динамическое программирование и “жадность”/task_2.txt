Сложность по времени: O(n)

Каждое число Фибоначчи F(k) для k=0,1,...,n вычисляется ровно один раз.

При последующих обращениях к F(k) результат берётся из кэша за O(1).

Всего нужно вычислить n+1 значений (F(0) до F(n)).

На каждое вычисление тратится O(1) операций (сложение двух предыдущих значений).

Итог: общее количество операций пропорционально n, то есть O(n).

Сложность по памяти: O(n)

Вектор memo имеет размер n+1, что требует O(n) памяти.

Глубина рекурсии в худшем случае равна n (когда алгоритм спускается от F(n) до F(0)), что добавляет O(n) памяти для стека вызовов.

Итог: суммарное потребление памяти составляет O(n)+O(n)=O(n).
